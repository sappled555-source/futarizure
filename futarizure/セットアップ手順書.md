# ふたりズレ v3.0 公開セットアップ手順書(初心者向け)

この手順書のとおりに進めると、**無料**で以下ができるようになります。
- 世界中どこからでもアクセスできるURLでアプリを公開(GitHub Pages)
- 離れた相手とのオンライン対戦(Firebase無料枠)
- スマホの「ホーム画面に追加」で、アプリのように起動(PWA)

作業時間の目安: 合計30〜40分。パソコンでの作業をおすすめします。

---

## 全体の流れ(WBS 4.5 に対応)

```markdown
# WBS 4.5 公開作業(使用者)
- STEP 1. Firebaseプロジェクト作成 ......... 約10分(オンライン対戦の土台)
- STEP 2. 設定コードをindex.htmlに貼る ..... 約5分
- STEP 3. GitHubアカウント作成&公開 ....... 約15分(アプリのURLができる)
- STEP 4. スマホで動作確認&ホーム画面追加 .. 約5分
```

※ オンライン対戦を後回しにしたい場合は、STEP 1〜2 を飛ばして STEP 3 から始めてもOKです。その場合も「この端末で遊ぶ」モードとPWAは動きます。

---

## STEP 1. Firebaseプロジェクトを作る(無料)

Firebaseは、Googleが提供するアプリの裏側(サーバー)サービスです。オンライン対戦の「部屋」のデータをここで中継します。

1. https://console.firebase.google.com/ を開き、Googleアカウントでログイン
2. 「プロジェクトを作成」→ プロジェクト名は `futarizure` など好きな名前 → Googleアナリティクスは**オフでOK** → 作成
3. 左メニューの「構築」→「**Realtime Database**」→「データベースを作成」
   - ロケーション: `asia-southeast1(シンガポール)` を選択
   - セキュリティルール: 「**テストモードで開始**」を選択 → 有効にする
4. 作成できたら「ルール」タブを開き、内容を下記に**書き換えて「公開」**を押す
   (テストモードは30日で切れるため、部屋データだけ読み書きできるルールにします)

```json
{
  "rules": {
    "rooms": {
      "$room": {
        ".read": true,
        ".write": true
      }
    }
  }
}
```

5. 左上の歯車 →「プロジェクトの設定」→ 下にスクロールし「アプリを追加」で **</>(ウェブ)** を選択
   - アプリのニックネーム: `futarizure-web` など → 「アプリを登録」
6. 画面に表示される `const firebaseConfig = { apiKey: "...", ... };` の **{ } の中身をコピー**しておく
   - ⚠️ もし `databaseURL` という行が含まれていない場合: Realtime Databaseの「データ」タブ上部に表示されている `https://〜.firebasedatabase.app` のURLをコピーし、`databaseURL: "そのURL",` という行を自分で追加してください(オンライン対戦に必須です)

## STEP 2. 設定コードを index.html に貼る

1. `futarizure-v3` フォルダの `index.html` をメモ帳(またはテキストエディタ)で開く
2. `★ セットアップ` と書かれたコメントのすぐ下にある、この行を探す:

```javascript
const FIREBASE_CONFIG = null;
```

3. `null` を、STEP 1-6 でコピーした内容で置き換える。完成形の例:

```javascript
const FIREBASE_CONFIG = {
  apiKey: "AIzaXXXX...",
  authDomain: "futarizure.firebaseapp.com",
  databaseURL: "https://futarizure-default-rtdb.asia-southeast1.firebasedatabase.app",
  projectId: "futarizure",
  storageBucket: "futarizure.appspot.com",
  messagingSenderId: "123456789",
  appId: "1:123456789:web:abcdef"
};
```

4. 保存する

※ このapiKeyは「秘密のパスワード」ではなく、Webアプリでは公開される前提の識別子です。上記のセキュリティルールでアクセス範囲を制限しています。

## STEP 3. GitHub Pagesで公開する(無料)

1. https://github.com/ でアカウントを作成(無料プランでOK)
2. 右上の「+」→「New repository」
   - Repository name: `futarizure`
   - **Public** を選択 → 「Create repository」
3. 作成後の画面で「uploading an existing file」というリンクをクリック
4. `futarizure-v3` フォルダの中の **5ファイル全部**(index.html / manifest.webmanifest / sw.js / icon-192.png / icon-512.png)をドラッグ&ドロップ → 「Commit changes」
5. リポジトリの「Settings」タブ →左メニュー「Pages」
   - Branch: `main` / フォルダ: `/(root)` を選択 → 「Save」
6. 1〜2分待ってページを再読み込みすると、上部にURLが表示されます:
   `https://あなたのユーザー名.github.io/futarizure/`
   これがアプリの公開URLです!

## STEP 4. スマホで動作確認&ホーム画面に追加

1. スマホのブラウザで公開URLを開く
2. **iPhone(Safari)**: 共有ボタン →「ホーム画面に追加」
   **Android(Chrome)**: メニュー(⋮)→「ホーム画面に追加」または「アプリをインストール」
3. ホーム画面にベン図アイコンの「ふたりズレ」が追加されればPWA化成功!
4. オンライン対戦の確認: 自分のスマホで「部屋をつくる」→ 相手に招待リンクを送る → 相手が自分のスマホで参加 → お互いの画面で回答→めくりが同期すればOK

## 動作確認チェックリスト

- [ ] 公開URLでアプリが開く
- [ ] 「この端末で遊ぶ」が今までどおり動く
- [ ] ホーム画面に追加でき、アイコンから起動できる
- [ ] 部屋をつくると4文字の合言葉が表示される
- [ ] 別のスマホから合言葉/リンクで参加できる
- [ ] ホストがお題を出すと、全員の画面にお題が出る
- [ ] 全員が回答を送ると、答え合わせ画面に切り替わる
- [ ] カードをタップすると、**全員の画面で同時に**めくれる

## うまくいかないときは

| 症状 | 確認すること |
|---|---|
| 「オンライン対戦はまだ準備中です」と出る | STEP 2 の貼り付けができているか(`null` のままになっていないか) |
| 部屋が作れない/参加できない | STEP 1-6 の設定に `databaseURL` の行があるか。STEP 1-4 のルールを「公開」したか |
| ホーム画面に追加してもアイコンが出ない | URLが `https://` で始まっているか(GitHub Pagesなら自動でhttpsです) |
| 更新したのに変わらない | PWAは古い画面を記憶します。ブラウザの再読み込みを2回、それでもダメならアプリを一度閉じて開き直す |

困ったら、エラー画面のスクリーンショットを添えてClaudeに相談してください。バグ修正はこちらで対応します。

## 既知の制限(現バージョン)

- 部屋のデータは残り続けるため、たくさん遊ぶとデータベースに古い部屋が溜まります(無料枠には十分収まる量ですが、次バージョンで自動掃除を実装予定)
- ゲーム中(お題が出た後)の途中参加はできません。ロビーに戻ってから参加してもらってください
